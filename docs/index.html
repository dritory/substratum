Substratum

an atlas of where the Standard Model and ΛCDM are under empirical stress, and the non-negotiable benchmarks any successor framework must clear.

σ‑over‑time, all tensions

Each line is one anomaly. Numeric σ is parsed from tension_history; hover for the original wording and the comparison being drawn. Multiple comparisons in the same year (e.g. data‑driven vs lattice SM) appear as separate points on the same line.

Per‑tension detail

Small multiples — one card per anomaly, with the same axes for legibility. Cards link to their source file on GitHub.

Length–energy landscape

Every entry placed on log‑log (ℓ, E) axes by its characteristic_scale (tensions) or regime (benchmarks). Circles are open tensions; squares are benchmark constraints. Color encodes layer and status. Click a marker for full detail.

Benchmark coverage

How the 67 benchmark constraints break down. Kind is the role a benchmark plays (reduction, precision test, principle invariance, forbidden phenomenon, cosmological observable, meta). Evaluator status is how hard it is to actually test a candidate framework against the benchmark in practice.

Framework evaluation

Every framework in frameworks/ scored against every benchmark in benchmarks/. The verdict comes from the per-benchmark evaluator under evaluators/: a closed-form compare for trivial bounds, an N-σ check for measurements with uncertainties, or open when the framework supplies no prediction. Hover any cell for the verdict note and the prediction that produced it.

Per-framework tally

Stacked counts of how each framework's predictions land. Open cells are unscored, not failed — a framework that has not yet supplied a prediction cannot be falsified by the benchmark, only ignored by it.

Coverage by physics category

Where each framework actually says something, and where it stays silent. Categories sorted by how many benchmarks the framework leaves open — the long bars at the top are the physics areas the framework currently doesn't address.

Verdict matrix

Rows are frameworks, columns are benchmarks (sorted by the fraction of frameworks that pass them — left columns are easy benchmarks that most frameworks clear, right columns are discriminating ones).

Browse all entries

One row per JSON file. Filter by layer, domain/kind, status, and evaluator status. Click any row for full detail and primary references.
